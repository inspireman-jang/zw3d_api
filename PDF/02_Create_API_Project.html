2장: ZW3D API 프로젝트 생성

1. 시스템 요구사항

📋 필수 시스템 요구사항

  • 운영체제: Windows 7 SP1 또는 Windows 10 또는 Windows 11 (Windows 10 권장)
  • 개발환경: Visual Studio 2019 (또는 C/C++용 다른 IDE)
  • ZW3D 버전: ZW3D 2023 이상 (현재 ZW3D 버전과 일치하는 것을 권장)
  • UI 개발: Qt5.9.7 (Windows용)

📝 참고: Qt 설치 후 4장의 방법을 참조하여 UI 편집 플러그인을 설치하면, ZW3D에서 제공하는 사용자 정의 컨트롤을 사용하여 ZW3D 플랫폼에서 UI 디자인을 완료할 수 있습니다.

2. ZW3D 템플릿으로 시작하기

복잡한 환경 구성 작업을 절약하고 ZW3D 2차 개발을 빠르게 수행할 수 있도록 ZW3D 템플릿을 제공합니다. api 폴더 내에서 "ZW3DTemplateVSIXProject.vsix"라는 Visual Studio 플러그인 설치 프로그램을 찾으세요.

a템플릿 설치

설치 프로그램을 더블클릭하면 로컬 Visual Studio 플랫폼을 자동으로 감지하고(2017, 2019, 2022 버전에 적응), 프롬프트에 따라 마법사 템플릿을 더블클릭하여 설치합니다. 설치에는 몇 분이 소요될 수 있습니다.

설치하려는 버전을 선택하고 Install을 클릭하여 해당 2차 개발 마법사 템플릿을 자동으로 설치합니다. 설치가 성공하면 해당 프롬프트가 표시됩니다.

b프로젝트 생성

Visual Studio 버전에 따라 템플릿을 선택하여 개발 프로젝트를 직접 생성합니다.

새 프로젝트 만들기 → 최근 프로젝트 템플릿 → Dynamic-Link Library (DLL) 선택

c프로젝트 구성

"HelloZW3DAPI"라는 개발 프로젝트를 생성합니다.

프로젝트 이름: HelloZW3DAPI
위치: D:\ZW3D\training\API_Training\
솔루션 이름: HelloZW3DAPI

d템플릿 구조

템플릿에는 편집용 프로젝트 템플릿과 테스트 케이스가 포함되어 있습니다.

Solution 'HelloZW3DAPI'
├── HelloZW3DAPI
│ ├── References
│ ├── External Dependencies
│ ├── inc
│ │ └── HelloZW3DAPIPr.h
│ ├── Resource Files
│ └── src
│ ├── CustomCommand.cpp
│ ├── FormCommand.cpp
│ ├── HelloZW3DAPI.def
│ ├── main.cpp
│ └── TemplateCommand.cpp

e테스트 케이스 확인

"CustomCommand.cpp"를 열어 제공된 테스트 케이스 "HelloZW3DAPI"를 확인할 수 있습니다.

f템플릿 제거

개발 도구를 제거해야 하는 경우, Visual Studio의 플러그인 관리를 통해 해당 템플릿 플러그인 "ZW3DTemplateVSIXProject"를 찾아 제거를 클릭하고 지침에 따라 개발 템플릿을 제거할 수 있습니다.

3. 환경 구성

컴퓨터에 ZW3D 프로그램이 설치되지 않았거나, 마법사 템플릿이 사용되지 않거나, 기타 이유로 엔지니어링 환경에 문제가 발생한 경우, 아래 지침에 따라 파일을 생성하고 환경을 구성할 수 있습니다.

a헤더 파일 생성

2차 개발 마법사 템플릿의 내용을 기반으로 "HelloZW3DAPI.h"를 생성합니다.

#ifndef HELLOZW3DAPI_H
#define HELLOZW3DAPI_H
#include "VXApi.h"

int HelloZW3DAPIInit(int format, void *data);
int HelloZW3DAPIExit(void);
int HelloZW3DAPI(void);

#endif

b소스 파일 생성

2차 개발 마법사 템플릿을 사용하여 "HelloZW3DAPI.cpp"를 생성합니다.

#include "HelloZW3DAPI.h"

int HelloZW3DAPIInit(int format, void *data)
{
    cvxCmdFunc("HelloZW3DAPI", (void*)HelloZW3DAPI, VX_CODE_GENERAL);
    return 0;
}

int HelloZW3DAPIExit(void)
{
    cvxCmdFuncUnload("HelloZW3DAPI");
    return 0;
}

int HelloZW3DAPI(void)
{
    cvxMsgDisp("Hello ZW3D API!");
    return 0;
}

c모듈 정의 파일 생성

2차 개발 마법사 템플릿의 내용을 기반으로 "HelloZW3DAPI.def"를 생성합니다.

LIBRARY HelloZW3DAPI.dll
EXPORTS
HelloZW3DAPIInit
HelloZW3DAPIExit
HelloZW3DAPI

d프로젝트에 파일 추가

"HelloZW3DAPI.h", "HelloZW3DAPI.cpp", "HelloZW3DAPI.def"를 프로젝트에 추가합니다.

e헤더 디렉토리 설정

프로젝트를 마우스 오른쪽 버튼으로 클릭하고 속성을 선택한 후, ZW3D API 헤더 디렉토리를 C/C++ → 일반 → 추가 포함 디렉토리에 추가합니다.

C:\Program Files\ZWSOFT\ZW3D 2025\api\inc;

참고: ZW3D 2024 이전 버전인 경우 포함 디렉토리에서 "inc"를 제거해야 합니다.

f라이브러리 디렉토리 설정

ZW3D API 라이브러리 파일 디렉토리를 링커 → 일반 → 추가 라이브러리 디렉토리에 추가합니다.

C:\Program Files\ZWSOFT\ZW3D 2025\api\lib;

g라이브러리 및 모듈 정의 파일 설정

ZW3D API 라이브러리를 링커 → 입력 → 추가 종속성에 추가합니다. 그런 다음 def 파일을 링커 → 입력 → 모듈 정의 파일에 추가합니다.

h리소스 컴파일 설정

리소스를 컴파일하려면 빌드 이벤트 → 빌드 후 이벤트 → 명령줄에 명령줄을 추가합니다.

IF EXIST "$(ZW3D_DIR)zrc.exe" "$(ZW3D_DIR)zrc.exe" "$(SolutionDir)." -o "$(TargetDir)$(ProjectName).zrc"

참고: 명령은 연속적인 한 줄의 내용이므로 두 줄로 처리하지 마세요. 리소스 파일이 없으면 추가하지 않아도 되며, "ZW3D_DIR"은 ZW3D 설치 경로 값을 가진 환경 변수입니다.

4. 프로젝트 생성

이 프로젝트를 마우스 오른쪽 버튼으로 클릭하여 프로젝트를 빌드합니다. 그러면 "...HelloZW3DAPI\x64\Release\HelloZW3DAPI.dll" 디렉토리에서 "HelloZW3DAPI.dll"을 찾을 수 있습니다.

✅ 빌드 성공 메시지:

1>Finished generating code
1>HelloZW3DAPI.vcxproj -> C:\ZW3D\training\API_Training\HelloZW3DAPI\x64\Release\HelloZW3DAPI.dll
1>Done building project "HelloZW3DAPI.vcxproj"
========= Build: 1 succeeded, 0 failed, 0 up-to-date, 0 skipped ==========

참고: "Release" 디렉토리는 프로젝트 구성에 따라 달라지며, "Debug"로 프로젝트를 빌드하면 "Debug"가 됩니다.

5. DLL 파일 로드

DLL 파일을 로드하는 여러 가지 방법을 제공합니다:

🔧 방법 1: ZW3D 애플리케이션 관리자 사용

ZW3D 애플리케이션 관리자를 사용하여 DLL을 로드하면 출력 대화상자에 명령이 실패했는지 성공했는지를 보여주는 메시지가 표시됩니다.

참고: 이 경로는 ZW3D가 닫힐 때 레지스트리에 기억되며, 다음에 자동으로 경로를 따라 파일을 로드합니다.

📁 방법 2: 사용자 디렉토리에 복사

HelloZW3DAPI.dll을 사용자 디렉토리의 "apilibs" 폴더에 복사하고 ZW3D를 실행합니다.

%appdata%\ZWSOFT\ZW3D\ZW3D2025\custom\apilibs

참고: ZW3D는 "apilibs" 폴더에서 애플리케이션을 검색하며, 있으면 로드됩니다.

🏢 방법 3: 설치 디렉토리에 복사

ZW3D 설치 디렉토리에 "apilibs" 폴더를 생성한 후 "HelloZW3DAPI.dll"을 폴더에 복사하고 ZW3D를 시작하면 이 폴더의 애플리케이션도 로드됩니다.

⚙️ 방법 4: 지정된 경로에서 리소스 파일 등록 및 로드 (권장)

이 방법은 2차 개발 플러그인에 더 권장됩니다.

a파일 배치

HelloZW3DAPI.dll 및 기타 리소스 파일을 사용자 디렉토리에 배치할 수 있습니다:

...\Desktop\ZW3D\ZW3D2025\mydll

b검색 경로 등록

플러그인 초기화 중에 "ZwResourcePathAdd"를 호출하여 검색 경로를 등록해야 합니다:

ZwResourcePathAdd("...\Desktop\ZW3D\ZW3D2025\mydll");

c레지스트리 설정

ZW3D를 열기 전에 레지스트리의 "HKEY_CURRENT_USER\Software\ZWSOFT\ZW3D"에 리소스 정보를 추가해야 합니다.

ZW3D 버전 29.00(2025)에 해당하는 플러그인 레지스트리 "ZW3D 29.00\en_US\Plugin_x64"에서 새 키를 생성하고 다음 내용을 추가해야 합니다:

  • Application: dll 파일의 이름
  • Description: 플러그인 설명
  • load: 플러그인 로딩 마크, 7로 설정
  • Location: dll이 위치한 폴더
  • Resource: 템플릿 명령 생성 시 추가해야 하는 zrc 파일의 완전한 경로

d다중 zrc 파일 등록

단일 dll에 대해 여러 관련 zrc 파일을 등록해야 하는 경우, ZwResourceFileRegister를 사용하여 필요한 zrc 파일을 등록할 수 있습니다. 이 인터페이스는 절대 경로를 통한 zrc 파일 로딩을 지원합니다.

⚠️ 중요: ZW3D가 활성화되면 레지스트리의 관련 정보를 자동으로 읽고 지정된 경로에서 플러그인과 리소스 파일을 로드합니다. 레지스트리 수정은 ZW3D가 활성화되지 않은 상태에서 수행해야 하며, 그렇지 않으면 변경사항이 적용되지 않습니다.

6. 애플리케이션 실행

명령줄에 "~HelloZW3DAPI"를 입력하고 Enter를 누르세요. 출력 대화상자에 "Hello ZW3D API!"가 표시되는 것을 확인할 수 있습니다.

💡 명령 이름 지정 팁: 함수 이름은 임의의 이름으로 정의할 수 있지만, 동일한 이름의 명령을 방지하기 위해 이름을 사용자 정의할 때 사용자 정의 사양이나 접두사를 사용하여 중복된 명령 이름으로 인한 등록 실패를 방지하는 것이 좋습니다. 함수를 등록할 때 반환값에 주의해야 하며, 이름이 이미 처리된 경우 인터페이스가 실패를 반환합니다.

7. 애플리케이션 디버깅

애플리케이션을 디버깅하는 두 가지 방법을 제공합니다:

🔍 방법 1: 프로젝트 디버그 명령 설정

프로젝트를 마우스 오른쪽 버튼으로 클릭하고 디버그 명령을 설정합니다. 로컬 Windows 디버거를 실행하거나 F5를 눌러 디버깅을 시작합니다.

설정 경로: 프로젝트 속성 → 디버깅 → 명령

ZW3D 실행 파일 경로를 지정합니다.

🔗 방법 2: 프로세스에 연결

ZW3D가 켜져 있고 dll 파일이 로드된 상태에서 디버깅의 프로세스에 연결을 통해 프로젝트를 ZW3D에 연결합니다.

연결 방법: 디버그 → 프로세스에 연결 → zw3d.exe 선택

8. 중단점 설정

main.cpp에 중단점을 설정합니다. 디버깅할 때 다음을 확인할 수 있습니다:

aDLL 로드 시

DLL이 로드되면 디버거가 HelloZW3DAPIInit()으로 들어갑니다.

b명령 실행 시

"~HelloZW3DAPI"와 같은 명령을 실행하면 디버거가 관련 함수로 들어갑니다.

c애플리케이션 제거 시

애플리케이션이 제거되면 디버거가 HelloZW3DAPIExit()로 들어갑니다.
