🔧 Q1. DLL 프로젝트를 어떻게 설정하나요?

✅ 단계별 가이드

1Visual Studio 2019에서 빈 DLL 프로젝트 생성

2다음 파일들을 프로젝트에 추가

파일 설명
VxApi.h ZW3D 제공 헤더 파일 (설치 경로의 api 폴더에 위치)
zw3d.lib ZW3D 제공 라이브러리 파일
XXX.def DLL 내보내기 함수 명시용 모듈 정의 파일
XXX.c / XXX.cpp 명령 구현용 소스 파일
XXX.tcmd ZW3D 명령의 리소스 파일
※ GUI가 필요 없는 명령이라면 생략 가능
XXX.ui ZW3D 스타일 UI (폼/다이얼로그) 파일
※ 필요 없으면 생략 가능
XXX.zcui 메뉴 등록용 파일
※ 메뉴 등록이 필요 없으면 생략 가능
XXX.png / .bmp 아이콘 이미지
zrc.exe .tcmd, .ui, .zcui, .png 파일을 .zrc로 컴파일하는 리소스 도구

3.def 파일을 링커 설정에서 명시

📦 Q2. 리소스 파일을 .zrc로 컴파일하는 방법은?

💡 명령어 예시:

cd "C:\ZW3D\install\folder" zrc.exe D:\project -o D:\project\MyAddon.zrc
  • zrc.exe는 지정된 폴더 내의 모든 .tcmd, .ui, .zcui, .bmp 등을 자동 검색
  • 출력 파일명(.zrc)은 DLL 이름과 동일해야 함 (MyAddon.dll → MyAddon.zrc)
📁 Q3. DLL 및 리소스 파일은 어디에 넣어야 ZW3D가 인식하나요?
  • ZW3D 설치 폴더 또는 사용자 폴더 아래 apilibs 폴더에 배치
  • apilibs 폴더가 없다면 수동으로 생성
🐞 Q4. DLL 프로젝트를 디버깅하려면?

🔹 방법 1: 실행 중 프로세스 연결

  • 메뉴 Debug > Attach to Process...
  • zw3d.exe 선택 후 "Attach" 클릭

🔹 방법 2: 프로젝트 설정에서 실행 경로 지정

  • 프로젝트 속성 > Debugging
  • "Command" 항목에 zw3d.exe 경로 입력
  • "Environment" 항목에 _No_Debug_Heap=1 입력
  • F5 키로 디버깅 시작
⚠️ Q5. 명령 실행 시 "No such symbol" 또는 "Not a valid command" 오류가 뜨는 이유는?

가능한 원인 및 해결 방법:

원인 해결 방법
① 초기화 함수 이름이 잘못됨 DLL 이름 + Init 형식으로 지정해야 함
예: DrawBox.dll → DrawBoxInit()
② DLL에서 함수가 export되지 않음 .def 파일에 함수 이름 명시 필수
③ 명령 등록 안됨 cvxCmdFunc() 사용하여 명령 등록 필요
④ 명령 호출 접두사 오류 .tcmd가 있는 명령은 !명령어, 일반 명령은 ~명령어 사용
⑤ .zrc 누락 또는 빌드 안 됨 zrc.exe로 다시 컴파일 후 apilibs 폴더에 복사
🧹 Q6. DLL 애플리케이션을 제거하려면?

ZW3D는 DLL을 동적으로 로드/언로드하므로,

DLL 및 관련 파일들을 삭제하면 제거 완료

📌 Q7. ZW3D는 동시에 몇 개의 DLL을 로드할 수 있나요?

최대 16개

📛 Q8. 등록된 명령어 이름이란?

cvxCmdFunc() 호출 시 사용하는 명령 키워드

예시:

cvxCmdFunc("CreateHole", (void*)ModelAddHole, VX_CODE_GENERAL);
항목 내용
"CreateHole" 등록된 명령어 이름 (UI, .tcmd에도 동일하게 사용됨)
ModelAddHole 실제 구현 함수 포인터
VX_CODE_GENERAL 명령 타입 코드 (일반 명령)
🔠 Q9. 명령어 이름 최대 길이는?

최대 15자 (ASCII)

예: CreateMySurface ✅, CreateFilletFace ❌

🧩 Q10. "Custom Feature"란 무엇인가요? 일반 피처와 차이는?

▶️ 정의:

두 개 이상의 피처를 하나로 결합한 사용자 정의 피처

예시 (.tcmd에서 설정):

<property name="function">CustomOp</property> <property name="custom">FilletBox.c</property>
  • FilletBox는 등록된 명령 이름
  • .c는 C 함수 구현을 의미

op_type 속성값:

값 의미
-1 히스토리에서 숨김 피처
0 미분류 피처 (기본값)
1 표시 속성 등 수정
2 지오메트리 수정 포함
-2 2와 유사하나 입력 엔티티 정리를 방지
3 삭제 피처
11 입력된 엔티티는 수정 안하나 외부 객체를 수정
12 고정 지오메트리로 재생성, 재정의 불가
🕘 Q11. 어떤 API는 히스토리를 생성하고, 어떤 API는 생성하지 않나요?
  • 엔티티를 수정하지 않는 API (예: 질의, 파일 생성 등)는 히스토리를 생성하지 않음
  • 반면, ZW3D GUI에서 히스토리를 생성하는 작업과 동일한 API는 히스토리 생성함
