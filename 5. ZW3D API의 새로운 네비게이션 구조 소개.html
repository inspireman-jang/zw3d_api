✅ 개요

ZW3D API는 기존 인터페이스를 기반으로 기능이 보강되고 체계화된 새로운 버전을 제공합니다.

기존 인터페이스는 변경되지 않고 유지되지만, 향후에는 새 인터페이스를 중심으로 발전합니다.

기존 API

  • cvx 접두어

신규 API

  • Zw 접두어
  • ezw 열거형(enums)
  • szw 구조체(structs)

🔍 새로운 API의 주요 특징

항목 설명
표준화된 명명규칙 Zw + 환경 + 객체 + 기능 (예: ZwPartFaceGetInfo)
줄임말 최소화 함수명, 구조체명에서 약어 사용 최소화
기능 중심 구조 기능별, 환경별로 인터페이스가 모듈화됨
기본 환경: Part 대부분의 기능은 Part 환경을 기본으로 지원하며, Assembly나 Sheet Metal 환경과 공통 사용 가능
Top-down 접근 필요한 기능 → 환경 → 관련 인터페이스 구조로 빠르게 검색 가능

🧩 신규 API 주요 모듈 요약

📂 1. Basic (기본 플랫폼 모듈)
  • 전역 변수 및 플랫폼 관리
  • 일반 기능 제공
🗂 2. File (파일 모듈)
  • ZW3D 파일 계층 구조 처리
  • 파일 열기, 저장, 루트 오브젝트 처리 등
  • 다중 루트 파일 지원 포함
🧱 3. Entity (객체 모듈)
  • Part, Shape, Wireframe, Component 등을 포함하는 일반화된 인터페이스
  • 다양한 객체 공통 처리
🔷 4. Brep (형상 토폴로지)
  • 토폴로지 객체 간의 관계 및 구조 처리
  • Shape, Face, Loop, PreEdge, Edge 등
🔳 5. WireFrame (와이어프레임)
  • Curve, Point 등 포함
  • 실재/가상 객체를 동일하게 취급
🧭 6. History (히스토리 트리)
  • 모델 히스토리 및 피처 정보 접근
  • 추후 히스토리 관리 전용 인터페이스 확장 예정
📐 7. Datum (기준 요소)
  • 기준축, 기준면 등의 생성 및 위치 지정
🔧 8. Feature (모델링 피처)
  • Shape, FreeForm, WireFrame 등 기본 모델링 피처 포함
  • Sheet Metal과 같은 특수 피처는 독립 모듈에서 관리
🏷️ 9. Attribute (속성 관리)
  • 객체 및 파일의 일반 속성
  • 단, 표시 속성(Display Attribute)은 객체별 모듈에 포함됨
📊 10. Equation (변수 및 수식)
  • 파라메트릭 모델을 위한 변수 접근 및 설정
  • Attribute와 연동하여 사용 가능
⚙️ 11. Parameter (파라미터 설정)
  • ConfigTable 등 단일 파일 내 다양한 파라메트릭 설정
➕ 12. Math (수학 연산)
  • Matrix, Vector, Bounding Box 등 수학 객체 및 연산 인터페이스 제공
🔄 13. DataExchange (데이터 교환)
  • Import, Export 관련 기능 및 설정
👁 14. View (뷰 설정)
  • 뷰 방향 및 디스플레이 제어
  • 주의: Drawing 뷰와는 별개 모듈
📝 15. PMI (제품 제조 정보)
  • Part 및 파생 환경에서의 표시 정보 처리
🧩 16. Assembly (조립 환경)
  • Assembly, Component, Constraint 등
  • 조립 환경에서 활성화된 상태에서 사용
📐 17. Drawing (도면 환경)
  • Sketch, 3D Sketch, Constraint, Annotation 등 도면 전용 인터페이스
  • Drawing 환경에서만 사용 가능
✏ 18. Sketch (스케치)
  • 2D 선 및 구속 조건 처리
  • 일부 일반 인터페이스는 해당 환경에서 사용 불가 → 별도 전용 API 필요

🔍 Inquire 모듈: 검색 전용 인터페이스

  • 여러 모듈에서 사용 가능한 범용 검색 인터페이스
  • 필요한 객체나 정보를 빠르게 조회할 수 있도록 구성

🖼️ UI 모듈 (Ui Module)

  • 사용자 인터페이스 및 디스플레이 관련 제어
  • UI 폼 관리, 사용자 상호작용 호출 기능 포함

✅ 신규 API 정리 예시

구분 기존 API 신규 API
함수 접두사 cvx Zw
열거형 접두사 evx ezw
구조체 접두사 svx szw
명명 규칙 줄임말 혼합 환경 + 객체 + 기능 (CamelCase)

📌 요약

새 API는 더 직관적이고 확장 가능한 구조를 따릅니다.

환경 기반 구조, 명확한 네이밍, 모듈화된 기능 구분으로 기존 API보다 학습과 유지보수가 용이합니다.

향후 ZW3D 애드온 개발은 가급적 이 새로운 API 체계를 기준으로 진행하는 것이 권장됩니다.
